Query Processing dalam Basis Data

Apa itu query?

Query adalah sebuah ekspresi bahasa yang menggambarkan data yang akan didapatkan kembali dari sebuah database. Query juga bisa diartikan sebagai kemampuan database pada komputer untuk menampilkan atau menyimpan informasi tertentu. Namun perlu diketahui bahwa kemampuan tersebut hanya berupa pre-defined code artinya jika permintaan tidak sesuai dengan informasi yang tersimpan di database, maka permintaan tidak dapat diproses. Umumnya, query dibagi dalam dua jenis yaitu :

  1. Select Query

    2. merupakan permintaan atas suatu informasi yang tersimpan dalam database.

  2. Action Query

    3. merupakan kegiatan updating (pembaharuan), insert (penambahan) maupun penghapusan sebagian atau keseluruhan informasi dalam database.

Bagaimana mengukur biaya sebuah query?

Optimizer query akan membuat informasi statistik yang tersimpan dalam katalog DBMS untuk memperkirakan besarnya biaya dari sebuah rencana query. Informasi yang tersimpan meliputi:

  1. Banyaknya baris data (record) dalam sebuah tabel
  2. Banyaknya blok berisi baris data dalam tabel
  3. Ukuran setiap baris data dari tabel (satuan byte)
  4. Banyaknya nilai unik dalam tabel untuk suatu atribut
Ada beberapa kemungkinan rencana evaluasi untuk sebuah query, dan itu penting untuk bisa membandingkan alternatif dalam hal biaya (perkiraan) dan memilih rencana terbaik. Untuk melakukannya, kita harus memperkirakan biaya operasi individu dan menggabungkannya untuk mendapatkannya biaya rencana evaluasi query.
Biaya evaluasi query dapat diukur dari banyaknya sumber daya (resource) sistem yang terpakai, meliputi pengaksesan disk, waktu CPU mengerjakan query, dan untuk sistem basis data paralel atau terdistribusi. Pada sistem basis data yang besar dapat memberikan kontribusi biaya query yang dominan. Perhitungan waktu CPU lebih sukar untuk diprediksikan, maka biaya pengeksekusian sebuah rencana query yaitu waktu yang dibutuhkan untuk pengaksesan ke disk.

Bagaimana sebuah sistem database mengevaluasi sebuah ekspresi?

Cara yang tepat untuk mengevaluasi ekspresi adalah dengan mengevaluasi satu operasi pada satu waktu, dalam urutan yang sesuai. Hasil dari setiap evaluasi adalah materialized Untuk mengevaluasi sebuah query,tidak hanya membutuhkan pembuatan ekpresi aljabar relasional, tetapi juga mengetahui lebih jauh bagaimana hasil evaluasi dari setiap operasi yg dapat dilakukan. Metode evaluasi query ada dua yaitu:

  1. Materialization
    2. hasil dari setiap operasi perantara dibuat dan kemudian digunakan untuk evaluasi dari operasi tingkat berikutnya. Biaya evaluasi tidak hanya mencakup biaya operasi yang terlibat. Saat menghitung perikiraan biaya algoritma, biaya penulisan hasil operasi ke disk diabaikan. Untuk menghitung biaya evaluasi ekspresi , harus menambahkan biaya semua operasi, serta biaya penulisan hasil perantara disk.
  2. Pipeline
    3. Dengan menggunakan evaluasi ini, dapat meningkatkan efisiensi evaluasi query dengan mengurangi jumlah file sementara yang dihasilkan. Mencapai pengurangan dengan menggabungkan beberapa operasi rasional ke dalam jaringan pipeline of operation. Dimana hasil dari satu operasi dilalui bersamaan dengan operasi berikutnya di pipeline

Query Processing dalam Basis Data

bagaimana sebuah sistem database mengevaluasi multiple ekspresi?

Database Management System (DBMS) menyediakan platform dan efisien untuk platform yang akan menggunakan ekspresi. Ekspresi adalah cara yang baik untuk memodelkan kepentingan pengguna dalam data yang diharapkan. Multiple ekspresi akan membuat proses penanganan kesalahan lebih sulit ketika kita perlu mengarahkan ulang baris yang salah karena kesalahan disebabkan oleh salah satu ekspresi dan lebih sulit untuk mengetahui ekspresi mana yang menyebabkan kesalahan.
Telah dicatat bahwa kemampuan untuk mengelola Ekspresi dalam manajemen database system (DBMS) akan sangat berguna:

  1. Manajemen Ekspresi dalam DBMS akan memfasilitasi penanganan Ekspresi yang efisien dengan menggunakan teknik seperti pengoptimalan query dan pengindeksan untuk menangani sejumlah besar Ekspresi secara efisien
  2. Penggabungan dukungan untuk mengekspresikan jenis data di DBMS memfasilitasi pengelolaan sejumlah besar Ekspresi dengan cara yang efisien dan terukur
Dengan subspansi pengguna dimodelkan sebagai ekspresi, setiap item data baru, sebuah publikasi baru, disesuaikan dengan semua ekspresi untuk menemukan ekspresi yang mengevaluasi yang benar. Perhatikan bahwa model pencocokan satu item data ini dengan banyak ekspresi di dalam model DBMS tradisional di mana satu query dievaluasi terhadap banyak baris. Saat menggunakan tipe data ekspresi, setiap item data (yang sesuai dengan baris dalam aplikasi DBMS) dievaluasi terhadap banyak ekspresi yang disebut sebagai inverted data-query paradigm.
